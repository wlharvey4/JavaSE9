B.3.1.9 Using the Link Option

You use -link option to classes referenced to by your code, but not documented in the current javadoc command run.

For links to go to valid pages, you must know where those HTML pages are located and specify that location with the extdocURL option. This allows third-party documentation to link to Java.

Omit the -link option when you want the javadoc command to create links only to APIs within the documentation it’s generating in the current run. Without the -link option, the javadoc command doesn’t create links to documentation for external references because it doesn’t know whether or where that documentation exists.

The -link option can create links in several places in the generated documentation. See Javadoc Doclets.

Another use is for cross-links between sets of packages: Execute the javadoc command on one set of packages, then run the javadoc command again on another set of packages, creating links both ways between both sets.
