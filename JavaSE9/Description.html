A.5.1.1 Description

The javadoc command parses the declarations and documentation comments in a set of Java source files and produces a corresponding set of HTML pages that describe (by default)

  • the public and protected classes,
  • nested classes (but not anonymous inner classes),
  • interfaces,
  • constructors,
  • methods, and
  • fields.

You can use the javadoc command to generate the

  • API documentation or
  • the implementation documentation

for a set of source files.

You can run the javadoc command on entire packages, individual source files, or both. When documenting entire packages, you can use either the -subpackages option to recursively traverse a directory and its subdirectories, or to pass in an explicit list of package names. When you document individual source files, pass in a list of Java source file names.
