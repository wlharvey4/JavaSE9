A.1.7 {@index}

The {@index} tag specifies an item that will be indexed and returned, and thus found when using the search feature added by JDK 9. It has the following syntax: 

{ @index ‘term usage-str’ }

Here, ‘term’ is the item (which can be a quoted string) to be indexed; ‘usage-str’ is optional. Thus, in the following @exception tag, {@index} causes the “error” to be added to the index: 

@exception IOException On input {@index error}

Note that the word “error” is still displayed as part of the description. Now it is also indexed. If you include the optional ‘usage-str’, then that description will be shown in the index and in the search box to indicate how the term is used.
