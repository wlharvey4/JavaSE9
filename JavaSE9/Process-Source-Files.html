A.6.1.3 Process Source Files

The javadoc command processes files that end in the source file extension and other files described in Source Files. If you run the javadoc command by passing in individual source file names, then you can determine exactly which source files are processed. However, that isn’t how most developers want to work because it’s simpler to pass in package names.

The javadoc command can be run three ways, without explicitly specifying the source file names.

  1. You can pass in package names,
  2. use the -subpackages option, or
  3. use wild cards with source file names.

In these cases, the javadoc command processes a source file only when the file fulfills all of the following requirements:

  • The file name prefix (with .java removed) is a valid class name.
  • The path name relative to the root of the source tree is a valid package name after the separators are converted to dots.
  • The package statement contains the valid package name.
